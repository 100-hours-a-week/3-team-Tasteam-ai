Continous batch

250109 제안

고정된 배치 크기 x -> 그저 들어와서 처리의 반복, 배치는 병렬로 작동하지만, 각각에 대해서도 병렬이라서 병렬로 일한다고 보면 됨.
-> 쉽게 말해서 각자 처리된다.

예시

[요청 흐름]          [GPU 내부 상태]
───────────────────────────────────────────────
t=0s  요청A 도착 ─▶ GPU 실행 (Slot 1)
t=0.1s 요청B 도착 ─▶ GPU Slot 2에 합류
t=0.2s 요청A 완료 ─▶ Slot 1 해제 → 요청C 합류
t=0.3s 요청D 도착 ─▶ Slot 3 추가

일반적인 배치 크기와의 비교

[일반 배치]
GPU time ────────────────▶
│■■■■ idle ■■■■ idle ■■■■■■■■ idle...

[Continuous batch (vLLM)]
GPU time ────────────────▶
│■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■
(요청이 중간에 들어오면 즉시 join)
