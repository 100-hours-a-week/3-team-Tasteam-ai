좋은 질문이네. 이건 **“1 run 기준 timeout”이냐, “sweep 전체 timeout”이냐**에 따라 다르게 잡아야 해.

---

## ✅ 1️⃣ 전제 정리

* `run_cap = 50`
* 1 run = **20 ~ 40분**
* worst case = 40분

---

# 🎯 경우 1: "각 run"의 timeout을 설정하는 경우

wandb agent 실행할 때,
보통 timeout은 **1 run 기준**으로 잡아.

### 🔹 평균 30분

### 🔹 worst 40분

### 🔹 약간 여유 포함

👉 권장:

```
60분 (3600초)
```

### 이유

* 데이터 로딩 지연
* GPU warmup
* 초반 epoch 불안정
* sweep scheduler 지연

이런 걸 감안하면 **40분 딱 맞추면 위험**함.

---

## ✔ 안전한 설정

| 상황        | timeout |
| --------- | ------- |
| 빡빡하게      | 45분     |
| 안정적으로     | 60분     |
| GPU 대기 포함 | 70~80분  |

**나는 60분 추천.**

---

# 🎯 경우 2: sweep 전체 시간 계산

최악 기준:

```
50 runs × 40분 = 2000분
= 약 33시간
```

현실적으로 평균 30분이면:

```
50 × 30 = 1500분
= 약 25시간
```

---

## 🚨 GPU 1개로 돌릴 경우

**순차 실행이면 하루 이상 걸림.**

---

## 🚀 GPU 2개 병렬이면

```
25시간 ÷ 2 ≈ 12~13시간
```

---

# 🎯 결론 정리

### ✅ run timeout:

```
3600초 (1시간)
```

### ✅ sweep 전체 소요:

* 25~33시간 (1 GPU 기준)

# 💡 추가로 생각할 것

만약 네가 DeepFM sweep이라면:

* epoch 줄이고
* early stopping 넣고
* validation 기준 prune 걸면

👉 평균 runtime 20~25분까지 줄일 수 있음.

그러면 전체 16~20시간 컷 가능.
