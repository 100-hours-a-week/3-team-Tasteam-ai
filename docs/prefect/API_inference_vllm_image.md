## 이미지 역할이 다름

- **학습 이미지** (예: `train-llm`): 학습·merge 스크립트 등이 들어 있고, **학습 Pod / merge Pod**에서만 사용.
- **vLLM 이미지**: **추론(서빙)** 전용. vLLM 서버가 올라가고, 마운트된 볼륨의 **특정 경로**에서 모델을 읽어서 API로 제공.

추론은 **vLLM 이미지**로 떠 있는 Pod에서만 돌아가야 하고, 학습 이미지로는 vLLM 서버를 띄우지 않으니까 **추론에 학습 이미지를 쓸 수 없습니다.**

---

## 정리하면

1. **추론 = vLLM 이미지** → 그 이미지의 Docker CMD가 기대하는 **볼륨·경로**가 정해져 있음.
2. **학습/merge = 학습 이미지** → 학습용 볼륨(`4rlm64f9lv` 등)에 `runs/`, merge 결과 등을 씀.
3. merged 모델을 **추론에 쓰려면**  
   - **vLLM(추론) Pod가 마운트하는 볼륨**에,  
   - **vLLM CMD가 읽는 경로/구조**로 올려야 하고,  
   - 그건 학습용 볼륨·레이아웃과는 다를 수 있음.

그래서 “추론은 vLLM 이미지에서 실행돼야 하니까 학습 이미지는 못 쓴다”는 말씀이 맞고, **추론용은 vLLM 이미지 + vLLM이 쓰는 추론용 볼륨/경로**로 가져가야 합니다.
