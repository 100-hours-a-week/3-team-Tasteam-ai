## 2. 끌어올리기 방향 (우선순위)

### (1) evidence_validity 끌어올리기

- **출력 형식/스키마 명시**
  - 프롬프트에 명확히:  
    `overall_summary`에는 **`summary`만** 넣고, `bullets`/`evidence`는 **절대 넣지 말 것**.  
  - “JSON만 출력하고, 마크다운 코드블록(```json) 없이 한 개의 객체만 출력” 강조.
- **인덱스 규칙 강화**
  - “evidence는 **해당 카테고리 리뷰 배열 길이 미만**의 0-based 정수만 사용.  
    **각 bullet 하나당 정확히 하나의 인덱스.** 배열 길이를 넘는 인덱스는 사용 금지.”
- **후처리 보강**
  - 파싱 후 `overall_summary` 안에 `bullets`/`evidence`가 있으면 **제거**하고 `summary`만 남기기 (676번 같은 케이스 방지).
  - `pred`가 ```json ... ``` 형태로 나오면 **블록 안의 JSON만** 파싱하도록 해서, 2961처럼 꼬인 구조가 들어가도 최대한 정상 필드만 사용.
- **학습/데이터**
  - “bullet ↔ evidence 인덱스”가 **의미적으로 맞는** 예제 비중을 늘리면, 인덱스만 맞추는 수준을 넘어서 점수 상승에 도움.

### (2) faithfulness 끌어올리기

- **프롬프트**
  - “**입력으로 준 리뷰 문장에 나온 메뉴·가게·분위기만** 요약할 것.  
    리뷰에 없는 메뉴명·직원 설명·다른 가게 스타일을 넣지 말 것.”
- **Few-shot**
  - “쌀국수집 / 중식당 / 루프탑 바”처럼 **서로 다른 가게 타입** 예제를 넣어서,  
    “리뷰에 나온 것만 요약”하는 패턴을 명시적으로 보여주기.
- **모델/용량**
  - 0.5B에서 “다른 가게 템플릿”을 많이 쓰는 한계가 있으면,  
    같은 데이터로 더 큰 모델을 쓰거나, **카테고리·가게 타입이 분명한 데이터** 비중을 늘리는 방향 검토.

### (3) category_correctness 끌어올리기

- **카테고리 정의를 문장 단위로 명시**
  - 예:  
    “**service**: 직원, 서비스, 대기, 매장 분위기, 쾌적함.  
     **price**: 가격, 가성비, 양, 비싸다/저렴하다, 가격 대비 만족.  
     **food**: 음식, 메뉴, 맛, 요리, 특정 메뉴명.  
     한 카테고리 내용을 다른 카테고리 필드에 넣지 말 것.”
- **Few-shot**
  - “price에 ‘직원이 친절해요’를 넣은 예”를 **잘못된 예**로 하나 넣고,  
    같은 내용을 “service에 넣은 올바른 예”로 고쳐서 대비시키기.
- **학습 데이터**
  - service/price/food 경계가 애매한 샘플을 정리하고,  
    “가격만 말한 문장 → price”, “맛/메뉴만 → food”처럼 **경계가 분명한 샘플** 비중을 늘리기.

### (4) 공통

- **폴백과 “채우기” 규칙**
  - “리뷰가 1개 이상 있는 카테고리는 **반드시** summary + bullets + evidence를 채울 것”은 이미 있으니 유지.
  - “**price도** 리뷰가 있으면 bullets/evidence를 0으로만 두지 말고, 해당 price 리뷰 인덱스를 사용할 것”을 한 줄로라도 추가하면,  
    price에서 `[0,0,0,0]`만 나오는 패턴을 줄이는 데 도움이 됨.
- **Judge 루브릭**
  - “evidence가 모두 0이면 bullet–evidence 미준수로 감점”을 v2 루브릭에 이미 두었다면 유지.  
    필요하면 “의미적 불일치(해당 인덱스 리뷰가 bullet을 지원하지 않음)도 evidence_validity 감점”을 명시해 두면,  
    학습/데이터 개선 방향과 맞출 수 있음.

## 3. 요약: 무엇을 먼저 할지

- **즉시 적용 가능 (코드/프롬프트)**  
  - overall_summary에 `bullets`/`evidence` 금지 문구 + 파싱 후 제거.  
  - evidence “해당 카테고리 배열 길이 미만, bullet당 1인덱스” 문구 강화.  
  - 카테고리 정의(service/price/food) 문장 + “다른 카테고리 내용 넣지 말 것”.  
  - “리뷰에 나온 것만 요약” + “price도 리뷰 있으면 evidence 채울 것” 한 줄씩 추가.
- **단기 (데이터/학습)**  
  - bullet–evidence 의미 일치 예제 보강, 카테고리 경계가 분명한 샘플 비중 늘리기.  
  - Few-shot에 “가게 타입별 예시” + “잘못된 카테고리 예시 → 수정 예시” 추가.
- **중기**  
  - 같은 설정으로 더 큰 모델로 평가해 보거나,  
    소형 모델이면 “템플릿 오용”을 줄이기 위한 데이터 정제·재라벨링 검토.

이 순서로 적용하면 evidence_validity → faithfulness → category_correctness 순으로 개선되면서, **avg_score 3.0+**에 더 가까워질 수 있습니다.
