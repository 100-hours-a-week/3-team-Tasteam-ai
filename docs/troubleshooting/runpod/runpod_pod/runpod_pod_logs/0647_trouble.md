### 문제 1: Sentiment 400 (max_tokens 948)

- **에러**: `max_tokens too large: 948. ... your request has 3205 input tokens (948 > 4096 - 3205)`.
- **의미**: 입력 3205면 허용 출력은 **891**인데, 클라이언트가 **948**을 보냄 → 400.
- **상황**: 0533/0606 대비 cap이 적용된 상태(1024 대신 948 전송). 다만 **추정이 여전히 실제보다 작아서** 948이 서버 허용값 891보다 큼.
- **원인**: `_estimate_input_tokens`가 (total*2)//3일 때, 해당 메시지 길이로는 추정이 ~2892 수준인데 실제 입력은 3205. cap = 4096−2892−256 = 948 → 948 전송 → 서버 기준 891 초과.

### 문제 2: Summary 배치 400 (max_tokens 540)

- **에러**: `max_tokens too large: 540. ... your request has 3560 input tokens (540 > 4096 - 3560)`.
- **의미**: 입력 3560이면 허용 출력은 **536**인데, **540** 전송 → 400.
- **상황**: 4731 초과는 없음. 대신 **max_tokens가 허용값보다 4만큼 큼**.
- **원인**: 입력 토큰 추정이 실제(3560)보다 작게 나와 cap이 540으로 계산됨. 추정을 조금만 더 보수적으로 하면 536 이하로 내려감.
