## 1. 원인 정리 (0415 로그)

- **모델 max context = 4096** 인데, `max_tokens`(2048·1500) + **input tokens** 가 4096을 넘어서 400 발생.
- 일부는 **input만 4731** 로 4096을 넘는 경우도 있음.

---

## 2. 수정 사항

### `src/config.py`

- **`LLM_MAX_CONTEXT_LENGTH`** 추가 (기본 4096, env `LLM_MAX_CONTEXT_LENGTH` 로 변경 가능).

### `src/llm_utils.py`

- **`_estimate_input_tokens(messages)`**  
  - 메시지 전체 글자 수를 4로 나눠서 대략적인 input 토큰 수 추정.
- **`_cap_max_tokens_for_context(max_new_tokens, messages)`**  
  - `min(요청값, LLM_MAX_CONTEXT_LENGTH - 추정 input - 64)` 로 **max_tokens 상한** 계산.
- **`_call_runpod_openai`** / **`_call_runpod_openai_async`**  
  - RunPod 호출 시 위에서 구한 값으로 **max_tokens를 캡**해서 전달.

그래서 input이 길어도 **남은 컨텍스트 안에서만** max_tokens가 요청되도록 했습니다.
